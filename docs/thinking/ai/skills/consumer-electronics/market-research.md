# 市场调研

[← 回到「消费电子产品经理」](index.md)

做消费电子最上游的一个问题：**这个品类值不值得进、进去做什么**。我把"调研一个品类 → 推导出一个有依据的产品方案"做成了这个 Skill，让 Agent 按一条固定流水线产出报告。

## 设计思路

<div class="skill-idea">核心理念：<b>最终的产品方案不是凭空设计，而是从调研证据里长出来的</b>——差异化来自竞品差评与未满足需求，定价来自单位经济与竞品价格带，市场规模来自自下而上的测算。每一条结论都要能追溯到证据。</div>

整个调研按 7 个阶段推进，边搜边写、边记来源：

<div class="pipe">
  <span class="pipe-step">1 · 市场总览<em>规模/增速/格局 + 为什么涨跌</em></span>
  <span class="pipe-step">2 · 使用场景<em>谁在用、场景兴衰</em></span>
  <span class="pipe-step">3 · 品类×价格<em>分布图找空白/拥挤区</em></span>
  <span class="pipe-step">4 · 目标区间深挖<em>竞品 + 差评痛点</em></span>
  <span class="pipe-step">5 · 技术<em>脉络/趋势/IP 风险</em></span>
  <span class="pipe-step">6 · 供应链成本<em>BOM + 单位经济</em></span>
  <span class="pipe-step pipe-gate">◆ 数据校验关卡<em>交用户拍板</em></span>
  <span class="pipe-step">7 · 产品方案<em>功能/定价/TAM·SAM·SOM</em></span>
</div>

关键的一步是中间那个**校验关卡**：市场数据跑完先停下来交给用户确认或修正，用户拍板之后才生成产品方案——避免拿着一个我自己都没核实的数字去做决策。贯穿全程还压着几条硬纪律：

<div class="disc">
  <span class="disc-item"><b>口径钉死</b>品类/指标/地域/年份四维一致，出货量≠销量≠销售额，绝不混用</span>
  <span class="disc-item"><b>数字配归因</b>任何涨跌都要解释机制，区分"结构性驱动"与"一次性因素"</span>
  <span class="disc-item"><b>三角验证</b>关键数字交叉 2–3 个来源，打架就给区间</span>
  <span class="disc-item"><b>论文式引用</b>每个数字带 <code>[n]</code> 来源 + 高/中/低置信度</span>
  <span class="disc-item"><b>分地区看</b>中/美/欧/日韩分开，规模、合规、渠道都不混成一锅</span>
</div>
