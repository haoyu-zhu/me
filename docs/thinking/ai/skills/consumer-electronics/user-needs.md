# 用户需求

[← 回到「消费电子产品经理」](index.md)

**单品视角、自下而上**——深挖一款具体产品为什么这样、服务了谁、怎么演化。

## 设计思路

<div class="skill-idea">核心理念：<b>一个产品是一组被冻结的妥协</b>——它为某些用户的某些场景做了优化，必然牺牲了另一些。我们逆向地把这组妥协拆开，搞清楚它服务了谁、解决了什么真实痛点、为什么这样设计、为什么赢或输、往哪里走。</div>

分析按 7 层因果链推进，每一层显式引用上一层的结论：

<div class="pipe">
  <span class="pipe-step">1 · 场景 × JTBD<em>为完成什么任务雇它</em></span>
  <span class="pipe-step pipe-key">2 · 用户声音 → 痛点<em>命根子</em></span>
  <span class="pipe-step">3 · 卖点对症否<em>需求错配</em></span>
  <span class="pipe-step">4 · 竞品输赢<em>多因素归因</em></span>
  <span class="pipe-step">5 · Trade-off<em>为啥没法全都要</em></span>
  <span class="pipe-step">6 · 演化<em>妥协被逐步解除的历史</em></span>
  <span class="pipe-step">7 · 趋势<em>曲线 + 未满足痛点外推</em></span>
</div>

这个 Skill 最容易退化成"几篇媒体评测的综述"——但评测人挑剔的点（音质）和普通用户在乎的点（夹耳、续航、配对）经常背离。所以我在第 2 层设了一道**门控**，这是它的命根子：

<div class="disc">
  <span class="disc-item"><b>规模化用户声音</b>先采集 100–300 条真实评价（电商差评/小红书/B站/Reddit），做主题+情感聚类</span>
  <span class="disc-item"><b>痛点带占比背书</b>"差评中 X% 提音质、Y% 提佩戴"，强度用频次说话，禁止拍脑袋</span>
  <span class="disc-item"><b>缺声音必须标注</b>只有评测、拿不到用户声音时，结论一律标"评测推断，非用户验证"，不得作为决策硬依据</span>
</div>
