# 竞品分析

[← 回到「消费电子产品经理」](index.md)

锁定了品类和价位之后，下一个问题是：**这个价位段里都有谁、各家怎么打、还有哪个位置没人占**。

## 设计思路

<div class="skill-idea">核心理念：<b>竞品分析的价值不在于把每家抄一遍参数，而在于看清这个价位段里"各家在赌什么"</b>，从而找到没人占住的位置。</div>

差异化要从两个坐标轴同时看，缺一不可：

<div class="axes">
  <div class="axis-card"><b>横向 · 同价位</b>每个产品凭什么卖？逼出每家"一句话杀手锏"与软肋，分清哪些是真差异、哪些只是同质化营销话术。</div>
  <div class="axis-card"><b>纵向 · 跨价位</b>往上、往下一档差在哪？多花/少花的钱买到了什么——这定义了"加价逻辑"与"下放机会"。</div>
</div>

我把分析维度做成**可选模块**，开工先让用户勾选、不默认全跑（也有"快速扫描 / 标准分析"套餐）：

<div class="pipe">
  <span class="pipe-step">① 基础层<em>功能矩阵/参数/外观</em></span>
  <span class="pipe-step">② 用户侧<em>爽点/痛点/画像</em></span>
  <span class="pipe-step">③ 技术侧<em>只深挖分歧点</em></span>
  <span class="pipe-step">④ 市场层<em>销售策略/份额</em></span>
  <span class="pipe-step">⑤ 系列 SKU<em>哪款最好卖</em></span>
  <span class="pipe-step">⑥ 横向差异<em>一句话杀手锏</em></span>
  <span class="pipe-step">⑦ 纵向差异<em>加价逻辑</em></span>
  <span class="pipe-step pipe-key">⑧ 空当识别<em>落点</em></span>
</div>

整条链的落点是**空当识别**：把竞品摆进格局图 / 价值曲线，找出没人占住、可攻击的空当，交给产品定义去接。
